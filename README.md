# ai-docs

Practical documentation for AI-assisted software development. Built from production experience, not theory.

## What This Is

A collection of guides, methodologies, and reference materials for working effectively with AI coding tools. Everything here has been tested on real projects — shipped PRs, approved designs, production code.

## Structure

Each folder is a self-contained topic:

```
ai-docs/
├── copilot-markdown-workflow/   ← Methodology for GitHub Copilot context management
└── ...                          ← Future topics
```

## Current Content

### `copilot-markdown-workflow/`

How to control what GitHub Copilot reads — and why it matters more than what you type in the prompt.

Covers the five context layers (from `copilot-instructions.md` through story-level context documents), how to write effective instructions, and a methodology for structured context documents that produce specific, constrained output.

Includes a presentation deck, speaker notes, and customization guides for adapting the material to any team.

## Philosophy

We're in the early days of AI-assisted development. The tools exist. The methodology doesn't — not formally. These documents capture what works, learned through trial and error. Craft, not science. Starting points, not standards.
